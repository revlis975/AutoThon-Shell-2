Overview

This document describes the structure of the EcommerceFailureSample objects used in the e-commerce failure samples dataset. Each record represents a single failed test execution and includes metadata, categorization, impacted layers, and raw logs.

  • Status: always FAIL
  • Allowed failure types (8): Backend Service Bug Data Integrity Issue Integration Timeout Authentication Failure Concurrency Bug Configuration Error External Dependency Failure Data Freshness Breach
  • Additional properties: disallowed

status string (enum) Yes Execution status (always FAIL in dataset) Allowed: FAIL

impacted_layers array Yes Application layers involved in the failure Items type: string
correlation_id string Yes Correlation identifier to tie logs across systems
failure_categorization_reasoning string Yes Textual reasoning why failure is mapped to failure_type
logs array Yes Chronological log lines captured during test Items type: string
